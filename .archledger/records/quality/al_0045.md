## Quality Requirement

Every compiled wiki note must pass lint validation without errors.

### Acceptance Criteria

- `wikimason vault lint` returns zero findings.
- Required frontmatter fields present: tags, topics, status, created, updated, sources, source_count, aliases.
- All wikilinks resolve to existing files.
- No credentials or secrets in tracked files.

### Implementation

- `lint.py` orchestrates `lint_rules.py`, `lint_links.py`, `lint_credentials.py`.
- `lint_payload()` returns `{"ok": bool, "findings": [...]}` for JSON consumers.
- Integrated into `ingest finish` pipeline.
